# Web Application Scanning with OWASP ZAP

## Project Overview
This project demonstrates web application vulnerability scanning using OWASP ZAP. The target application was OWASP Juice Shop, hosted locally using Docker.

## Objective
The objective of this project was to identify common web application security issues, analyze their risk levels, and document remediation steps.

## Tools Used
- OWASP ZAP
- OWASP Juice Shop
- Docker Desktop
- Java 17
- Windows Command Prompt

## Target
http://localhost:3000

## Methodology
1. Installed Java 17, Docker Desktop, and OWASP ZAP.
2. Ran OWASP Juice Shop locally using Docker.
3. Performed automated scanning using OWASP ZAP.
4. Reviewed alerts and captured evidence.
5. Generated a ZAP HTML report.
6. Documented findings and remediation steps.

## Key Findings
- Content Security Policy Header Not Set
- Cross-Domain Misconfiguration
- Timestamp Disclosure - Unix
- Modern Web Application Observation

## Outcome
This project provided hands-on experience with web application vulnerability scanning, alert analysis, and security reporting.
